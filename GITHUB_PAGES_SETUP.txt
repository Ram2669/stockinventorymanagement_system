üöÄ STEP 1: CREATE GITHUB REPOSITORY
================================================================================

### Create New Repository:
1. Go to https://github.com
2. Click the "+" button in top right corner
3. Select "New repository"
4. Fill in repository details:
   - Repository name: stockinventorymanagement_system
   - Description: SRI LAKSHMI ENTERPRISES Stock Management System
   - Visibility: PUBLIC (required for free GitHub Pages)
   - DO NOT check "Add a README file"
   - DO NOT check "Add .gitignore"
   - DO NOT check "Choose a license"
5. Click "Create repository"

### Note Your Repository URL:
Your repository will be at:
https://github.com/YOUR_USERNAME/stockinventorymanagement_system

Replace YOUR_USERNAME with your actual GitHub username.

üåê STEP 3: ENABLE GITHUB PAGES
================================================================================

### Enable Pages:
1. Go to your repository: https://github.com/YOUR_USERNAME/stockinventorymanagement_system
2. Click the "Settings" tab (top right of repository page)
3. Scroll down in left sidebar and click "Pages" (under "Code and automation")
4. Configure GitHub Pages:
   - Source: Select "Deploy from a branch"
   - Branch: Select "main"
   - Folder: Select "/docs" (VERY IMPORTANT!)
5. Click "Save"

### Wait for Deployment:
- GitHub will show a message: "Your site is ready to be published"
- Wait 5-10 minutes for initial deployment
- Refresh the Pages settings page to see the live URL

### Your Website URL:
Once deployed, your site will be available at:
https://YOUR_USERNAME.github.io/stockinventorymanagement_system

‚öôÔ∏è STEP 4: CONFIGURE API CONNECTION
================================================================================

Your GitHub Pages site needs to connect to your local backend server.

### Find Your Local IP Address:
# WINDOWS:
ipconfig
# Look for "IPv4 Address" (usually 192.168.x.x)

# macOS/LINUX:
ifconfig | grep "inet " | grep -v 127.0.0.1
# Look for your network IP (usually 192.168.x.x)

### Update API Configuration:
Edit the file: docs/config.js

Change this line:
PRODUCTION_API_URL: 'http://YOUR_LOCAL_IP:5001/api',

To your actual IP address, for example:
PRODUCTION_API_URL: 'http://192.168.1.29:5001/api',

### Push the Configuration Update:
git add docs/config.js
git commit -m "Update API configuration with local IP address"
git push origin main

# Wait 2-3 minutes for GitHub Pages to update

üîß STEP 5: CONFIGURE YOUR LOCAL SYSTEM
================================================================================

### Start Your Backend Server:

python app.py

# Server should start on: http://0.0.0.0:5001

### Configure Firewall:
Allow port 5001 through your firewall:

# WINDOWS (Command Prompt as Administrator):
netsh advfirewall firewall add rule name="SRI LAKSHMI API" dir=in action=allow protocol=TCP localport=5001

# LINUX:
sudo ufw allow 5001

# macOS:
# System Preferences ‚Üí Security & Privacy ‚Üí Firewall ‚Üí Options
# Click "+" and add port 5001

### Test Local API:
curl http://localhost:5001/
# Should return JSON with API information
